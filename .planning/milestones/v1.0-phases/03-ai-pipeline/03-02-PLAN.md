1. **Code fence stripping** (Pitfall 2 from RESEARCH.md): The `parse_response` function MUST strip code fences before json.loads(). Use:
   ```python

       text = text.lstrip("`").lstrip("json").strip().rstrip("`").strip()

2. **anthropic.Anthropic() instantiation**: Create the client INSIDE `call_claude()`, not at module level. This is required so tests can patch `scripts.generate_exercises.anthropic.Anthropic` cleanly (anti-pattern from RESEARCH.md).

4. **Section header emoji**: Use the exact Unicode characters from UI-SPEC. Do NOT use escape sequences that could render differently. Copy from UI-SPEC §Section Structure Contract:
   - `## 📖 文章 / Article`  (U+1F4D6)
   - `## 📚 词汇 / Vocabulary`  (U+1F4DA)
   - `## 🔗 表达 / Chunking Expressions`  (U+1F517)
   - `## ❓ 理解 / Comprehension`  (U+2753)

5. **render_markdown vocabulary format**: Note NO period after definition if the field already ends with one — but to be safe, always append `.` only if the definition doesn't already end with `.`. Simplest: just render definition as-is (Claude should punctuate properly per prompt instructions).

6. **render_markdown Q&A format**: All three lines CONSECUTIVE with no blank line between them. Blank line comes AFTER `**A (中):**` line, before the next Q block.

7. **No temperature parameter** in call_claude signature (keep simple; max_tokens=2048 is a reasonable default per CONTEXT.md discretion).
